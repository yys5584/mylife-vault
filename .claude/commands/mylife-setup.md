# /mylife-setup

이 슬래시 커맨드를 실행하면 사용자에게 순서대로 질문해서 `templates/` 안 마크다운을 채운다.

포지션: 응답의 1순위 근거는 사용자가 직접 입력한 문서. 사주·별자리는 운세 수준의 참고 톤이며 선택 섹션, 명시 요청 시에만 답변에 반영. 자세한 룰은 `CLAUDE.md`의 "사주·별자리 — 위치 정의" 참조.

## 실행 순서

1. 현재 상태 진단

   - `templates/` 디렉터리 안 모든 `.md` 파일을 빠르게 grep해서 `{{` placeholder 개수 카운트
   - 사용자에게 보고: "현재 X개 placeholder 비어있음. 어디부터 시작?"
   - 옵션 제시:
     - A) 처음부터 핵심 7개 인터뷰 (사주·별자리 제외, 50-80분)
     - B) 섹션 선택 (필요한 것만)
     - C) 사주·별자리만 (선택, 색깔 도구. 5-10분)

2. 인터뷰 진행 — `SETUP.md` 따르기

   - `SETUP.md`에 섹션별 질문 시퀀스가 정의돼 있다
   - 한 섹션씩 진행. 한 섹션 끝나면 사용자에게 "다음 섹션 진행? 잠시 멈춤?" 확인
   - 사용자 답 받으면 즉시 해당 템플릿의 placeholder를 Edit으로 치환
   - 섹션 3의 0번(우선순위 1위 + 진짜 이유 + 달성 후 그림) 절대 빠뜨리지 않는다. 이게 빠진 vault는 자기 분석·시스템·로드맵이 어디로 가려는지 위에서 작동을 못 한다. 시간 압박 있어도 0번부터 솔직하게

3. 사주 + 별자리 계산 — 선택 (명시 요청 시만)

   - 기본 흐름에서는 자동 실행 안 한다. 사용자가 "사주도 채울래" / "별자리도 돌려줘"라고 명시했을 때만:
     - `scripts/calc_saju.py` → `templates/saju.md`
     - `scripts/calc_zodiac.py` → `templates/zodiac.md`
   - Python 없으면 만세력(원광대) + astro.com 안내 + 수동 입력 요청
   - LLM이 사주 8자·행성 위치를 직접 추정하지 않는다. 항상 스크립트 또는 검증된 사이트 결과 사용

4. 인터뷰 톤

   - 분석가 톤 금지. 코칭 파트너 톤
   - 한 번에 1-2개 질문만. 답 받으면 다음
   - 사용자가 "모름" / "넘어가" 하면 placeholder 그대로 두고 다음 질문
   - 답이 짧으면 후속 질문으로 살 붙임 (단, 질문 1개씩)
   - 워딩 룰 — 추상 조어 안 쓴다(엔진·갈래·격차 같은 거), 구체 명사 + 구체 동사, "또한·이를 통해·따라서·뿐만 아니라" 안 쓴다, "본인" 안 쓰고 호칭 또는 "사용자"로

5. 저장 & 검증

   - 매 섹션 끝날 때마다 해당 .md 파일 저장
   - 모든 섹션 끝나면 cross-reference 일치 확인:
     - `philosophy.md`의 정체성 한 문장 = `life_compass.md`의 정체성 한 문장
     - `self_profile.md`의 강점 = `life_compass.md`의 강점 표
   - 안 맞으면 사용자에게 알리고 통일 제안
   - 주의: 사주의 약한 오행을 `life_os.md` 처방의 근거로 끌어붙이지 않는다. 처방은 사용자 자체 약점에서 도출

6. 완료 메시지

   - 채워진 파일 리스트 출력
   - "이제 어떤 LLM에서든 이 폴더를 컨텍스트로 받으면 너에 맞춰 답함."
   - 첫 사용 권유: "오늘 하나 자문해볼래? 큰 결정 / 매일 자문 / 충동 점검 중에서."

## 부분 갱신 모드

사용자가 이미 채워진 vault에서 `/mylife-setup`을 다시 실행하면:

- "전체 갱신? 특정 섹션만? 새 사건 추가?"
- 새 사건은 `relationship_protocol.md`에 새 섹션 추가, 기존 사건 보존
- 분기 갱신은 `roadmap.md` + `life_compass.md`의 분기 목표·5년 북극성 검토

## 분할 진행 가이드

답변 페이스에 따라 시간 차이가 크다.

| 페이스 | 필수만 | 전체 |
|---|---|---|
| 빠르게 (한 줄 답) | ~30분 | ~65분 |
| 보통 (생각하며) | ~50분 | ~105분 |
| 깊이 (사례·검증) | ~90분 | ~180분 |

AI가 사용자에게 첫 질문: "처음 채우는 거면 빠르게로 일단 다 채우고 나중에 분기 갱신 때 깊이 보강 추천. 어떤 페이스로 갈래?"

분할 제안:

- 1차 (10-30분): 섹션 0 (기본) + 섹션 2 일부 (자기) + 섹션 4 (시스템) — 즉시 가치
- 2차 (15-50분): 섹션 2 마무리 + 섹션 3 (철학) — 깊은 작업
- 3차 (10-30분): 섹션 5 (컴파스·로드맵) + 6·7 (해당 시)
- 4차 (15-60분, 선택): 섹션 8·9·10 (연애·투자·직업 도메인 확장)
- (선택, 관심 시) 섹션 1 (사주·별자리): 운세 색깔. 응답 근거 아님

각 차수 끝나면 "여기까지 저장. 다음 차수 언제 할래?" 묻고 종료.

## 자동 보강 (인터뷰 마지막)

수동 인터뷰 끝나면 AI가 자동으로:

1. 나이 자동 계산: self_profile.md 생년월일에서 나이 채우기

(선택, 사용자가 사주·별자리도 채웠고 명시 요청 시만)

2. 사주↔철학 색깔 코멘트: saju.md 일간 기질 + zodiac.md 태양궁 trait → philosophy.md 사주↔철학 섹션에 한 줄 색깔로만 (사용자 답을 덮어쓰면 안 된다)
3. 빅3↔자기 진단 일치·충돌 노트: self_profile.md "참고 색깔" 섹션에 추가 (본문 안 건드림)

금지: 부족 오행·원소를 life_os.md 처방의 근거로 끌어 붙이는 것. 처방은 사용자 자체 약점에서만 도출.

## 인터뷰 마지막 — 보고서 출력 (필수)

인터뷰가 끝나면 LLM이 자동으로 **셋업 완료 보고서**를 출력한다. 이 보고서는 사용자가 "아 나는 이런 사람이구나"를 느끼게 하는 게 목적이다. 정보 나열이 아니라 자기 거울 역할이다.

### 보고서 구조

분류별 카드 5~6개 + 종합 보고서 1개. 카드는 사용자가 채운 문서에 따라 가감:

- 🧠 자기 (`self_profile.md` 기반)
- 🎯 인생 철학 (`philosophy.md` 기반)
- ⚙️ 시스템 (`life_os.md` · `life_compass.md` 기반)
- 💞 관계 (`relationship_protocol.md` · `love_style.md` 기반, 해당 시)
- 💼 일·돈 (`career_style.md` · `investment_style.md` · `side_project_strategy.md` 기반, 해당 시)
- 🪞 종합 (모든 카드 종합)

### 각 카드 형식

1. 헤딩 (이모지 + 카테고리명)
2. callout 한 줄 (`>` 형식, 카드의 핵심을 한 줄로)
3. 데이터 박스 (숫자·키워드·날짜를 짧게)
4. 본문 산문 3~5문단 (아래 4단계 흐름)

### 카드 본문 4단계 흐름

각 카드 본문은 다음 흐름으로 풀어쓴다:

1. 강점·특성을 인정 ("나는 ~한 편이다")
2. 그게 어떻게 잘 작동했는지 (긍정 측면)
3. 그게 어떻게 잘 안 작동했는지 (부정 측면, 위험 신호)
4. 그래서 뭐가 필요한지 (한 줄 솔루션)

### 종합 보고서 형식

카드 다음에 종합 보고서 1개. 4 블록 구조:

```
### 한마디로 어떤 사람인가
한 문장 유형 + 평소 어떻게 움직이는지 5~6개 bullet

### 강점
1~5개

### 약점·위험 신호
1~6개

### 그래서 필요한 것
각 항목마다 "왜 이게 작동하는지" 한 줄씩 같이
```

### 문체 룰 (가장 중요)

이 룰을 어기면 보고서가 판결문처럼 읽혀서 사용자가 자기 이야기로 못 받아들인다.

1. **주어는 "나"**. "사용자는 ~다" 금지. → "나는 ~한 편이다 / 내 패턴은 ~에 가깝다"
2. **숫자는 데이터 박스에만**. 본문에 "자기인식 9, 자기조절 3" 같은 숫자 남발 금지. 본문은 풀어 쓴다.
3. **단문 임팩트 끊기 금지**. "모르면 못 본다. 나는 본다." 같은 영어식 구조는 한국어로 읽으면 어색하다.
4. **자연스러운 한국어 흐름**. 주어 → 목적어 → 동사. 단락 안에서 문장을 연결한다.
5. **판결문 톤 금지**. 자기 서술 흐름으로.
6. **지시대명사보다 명사 주어**. "이건 ~했다" → "이 능력은 ~로 작동했다"
7. **"다만" → "하지만"**, **"나한테" → "나에게"** 등 자연스러운 한국어 어휘 우선.
8. **단락 사이 빈 줄**로 호흡을 만든다.
9. **문장 절제**. 굳이 안 적어도 되는 문장은 빼고 결론으로 자연스럽게 도착하게.

### 본문에 절대 쓰면 안 되는 두 가지

다른 어휘는 사용자가 평소 쓰는 정도면 본문에 OK. 단, 다음 둘은 *반드시* 풀어쓴다.

**1. 분석 비유 (장면이 안 그려지는 추상 비유)**

| ❌ 본문에 ❌ | ✅ 풀어쓰기 |
|---|---|
| 거울 역할 | 상대 고민을 계속 들어주고 정리해주는 역할 |
| 내 자리가 비어 있다 | 내 마음은 관계 안에서 다뤄지지 않는다 |
| 정보가 한 방향으로 흐른다 | 나는 다 말하는데 상대는 자기 얘기를 숨긴다 |
| 압력솥 패턴 | 혼자 쌓다가 한 번에 터지는 흐름 |
| 안전선이 작동한다 | 나를 지키려고 잠깐 멀어진다 |

vault 안에서 시스템 라벨로 쓰는 비유(압력솥 등)도 *보고서 본문에서는* 풀어 쓴다. 라벨은 메모용, 본문은 읽힘용.

**2. 컴파운드 즉석 조어 (사전에 없는 합성어)**

| ❌ 절대 ❌ | ✅ 풀어쓰기 |
|---|---|
| 완성 동력 | 끝까지 가는 힘 |
| 의지의 진위 | 내가 결심한 게 진짜인지 가짜인지 |
| 외부 룰 작동 | 미리 정해둔 규칙이 자동으로 켜진다 |
| 시스템 위에서 굴러간다 | 미리 짜둔 구조가 나 대신 굴러간다 |
| 본질 동력 | 진짜 움직이는 이유 |
| 결정 압축 | 결정 한 줄로 줄이기 |

단어 두 개를 인위적으로 붙여 만든 조어는 사용자한테 안 읽힌다. **무조건 풀어쓴다.**

### 본문에 써도 되는 어휘

다음은 정착된 용어라 본문에 OK:
- 대중 친숙 용어: 회피형 애착 · ADHD · MBTI 유형명
- vault 룰 이름: 4-1-1 룰 · One Project Rule · 비협상 룰
- 임상 진단어 (사용자가 자주 사용): 자기조절 결핍 · 충동제어 부족 · 정서 조절 곤란
- 비즈·경제 어휘 (사용자가 자주 사용): 시장 자산 · 현금 흐름 · 변수

진단법: *이 단어를 사전에서 찾을 수 있나?* 있으면 OK. 즉석에서 만든 합성어면 ❌.

### 출력 직전 자기 점검 (필수)

보고서·카테고리 풀이를 출력하기 *직전*에 자기 점검한다. 하나라도 어기면 다시 쓴다.

- [ ] 주어 "나" — 본문에 "사용자는"이 없는가?
- [ ] 숫자 점수가 본문 산문에 끼어 있지 않은가? (박스에만)
- [ ] 분석 비유가 있는가? (거울 역할·내 자리 비어 있다·정보 한 방향·압력솥 등)
- [ ] 컴파운드 즉석 조어가 있는가? (완성 동력·의지의 진위·외부 룰 작동 등)
- [ ] 각 문장 읽었을 때 머릿속에 그림이 그려지는가?
- [ ] "X. Y. Z." 단문 끊기 임팩트 구조가 있는가?
- [ ] 판결문 톤("X는 Y다" 단정)이 아닌가?

체크리스트는 출력에 포함시키지 않는다. 출력 *전* 자기 점검 용도.

### 좋은 톤 예시

> 나는 스스로를 꽤 정확하게 보는 편이다.
> 이 능력은 일이나 콘텐츠에서는 큰 장점으로 작동했다. 내가 사람들에게 어떻게 보이는지, 어떤 말이 진심처럼 들리는지를 빠르게 잡아낼 수 있었기 때문이다.
>
> 하지만 삶에서는 이 능력이 늘 좋은 쪽으로만 작동하지 않는다. 나는 내가 무너지는 장면도 너무 잘 본다. 알면서도 멈추지 못하는 순간, 같은 실수를 반복하는 순간을 그냥 넘기지 못한다.
>
> 결국 나에게 필요한 건 더 강한 결심이 아니라, 결심이 약해지는 순간에도 굴러가는 구조다.

### 나쁜 톤 예시 (피할 패턴)

> 사용자는 자기를 잘 본다. 자기인식 9, 메타인지 9.
> 동시에 그 자각이 매일 약점도 보여준다. 모르면 못 본다. 사용자는 본다.
> 그래서 자기 비난이 깊고 피로도 높다.

(주어 "사용자", 숫자 남발, 단문 끊기, 판결문 톤. 한국어로 읽으면 난독)

### 인용 가능한 외부 레퍼런스 (필요 시만)

보고서가 사용자 입력을 일반 패턴과 비교하는 경우, 검증된 출처만 인용:

- 애착 유형 분포: Bartholomew & Horowitz 1991
- ADHD 성인 자기조절 갭: Barkley 2015 메타분석
- VIA 24 강점 분류: Peterson & Seligman 2004
- ACE 어린시절 역경: Felitti et al. 1998 (CDC-Kaiser)
- Big5 한국 규준: IPIP-K, Lim & Ham 2003
- 한국 30대 평균 자산: NABO 등 공공 통계

사용자 응답을 위 프레임의 언어로 매핑할 때는 반드시 "추정"임을 표시한다. 정식 검사 결과가 아니다.

### 보고서 다음 — 카테고리별 풀이 메뉴

보고서가 끝나면 사주에서 *총괄 풀이 → 운별 풀이*로 넘어가듯, 영역별 깊이 풀이로 들어갈 수 있게 메뉴 띄운다.

```
방금 본 건 전체 그림이다. 이제 영역별로 더 깊이 볼 수 있다.
하나 골라서 던지면 vault 위에서 풀어준다.

1. 나와 맞는 연인 유형 / 만나면 위험한 유형
2. 나와 맞는 직업·역할 / 피해야 할 직업
3. 나와 맞는 사이드 방향 / 시작 9·완성 1로 깨질 사이드
4. 내 돈 그릇·재물 패턴 / 잃을 가능성이 높은 영역
5. 친구·동료 관계에서 자주 빠지는 함정
6. 부모·가족 관계가 지금에 미치는 영향

(선택, 명시 요청 시만)
- 사주 일간 풀이 / 별자리 빅3 해석
- 올해·내년 운 (saju 세운·zodiac transit 기반)
```

### 카테고리 풀이 작동 룰

사용자가 메뉴에서 하나 고르면 LLM은 다음 흐름으로 답한다.

**1. 사주 같이 볼지 사용자에게 묻기**

`saju.md`·`zodiac.md`가 채워져 있으면 답하기 전에 한 번 묻는다:

```
사주(또는 별자리) 풀이도 같이 봐줄까?
사주가 내 답변이랑 같은 방향을 가리키는지, 다른 방향을 가리키는지 볼 수 있다.
```

채워져 있지 않으면 묻지 않고 vault 데이터로만 답한다.

**2. 사용자가 "응" 하면 두 풀이 같이 출력**

답변 형식:

```
## 💞 [카테고리명]

> [callout 한 줄]

[데이터 박스: 잘 맞는 사람 · 위험한 사람 · 시기 · 일치도]

---

### 📌 내가 채운 답변 기반

[vault 데이터로 풀이. 1인칭 산문]
**[하위 제목]**
1. **[항목 제목]**
   [짧은 산문 1~2 문장. 장면이 그려지는 문장]
2. **[항목 제목]**
   ...

### 📌 사주 풀이로 보면

[중요] 사주 풀이는 사용자 본인 기질만 풀지 않는다. 사용자가 "어떤 사람을 만나야 하나"를 물은 거면, 짝의 그림을 4축으로 풀어준다:

**짝의 성격**
[saju.md 일간·십신·오행 균형에서 도출. 일상어로 풀기]

**짝의 외모**
[정재(남)·정관(여) 등 배우자 자리 + 신살에서 도출. 일상어로 풀기]

**짝의 일·환경**
[인성(가정·배경) + 관성(외부 받침) 자리에서 도출]

**시기 신호**
[대운 + 세운에서 현재·앞으로 1~3년 신호]

직업·돈·가족 카테고리는 4축이 다르게 적용:
- 직업: 일간 기질 + 식상·관성 균형 + 어떤 일이 잘 맞는지 + 시기
- 돈: 재성 분포 + 비겁 충돌 + 어떤 돈이 잘 맞는지 + 시기
- 가족: 인성(부모)·비겁(형제) 자리 + 영향 패턴 + 회복 길

어휘는 "기질적 경향·강약 영역·잘 맞는·자연스럽게 끌리는"만. "운명 결정" 어휘 ❌.

### 📌 두 풀이가 만나는 곳

```
🟢🟢🟢🟢⚪  강한 일치 (4/5)
```

답변 풀이와 사주 풀이가 같은 축에서 비교되도록 정렬:
- 둘 다 짝의 *성격*에 대해 뭐라고 했나
- 둘 다 짝이 받쳐줘야 할 *내 약점*에 대해 뭐라고 했나
- 둘 다 *시기 신호*에 대해 뭐라고 했나
- (사주에만 있는) 외모·환경 같은 축은 *답변에는 명시 없지만 실제 끌림에 작동했을 가능성*으로 추가

**같은 사람을 가리킨다**
1. [축 1] ... 답변/사주 어휘 비교
2. [축 2] ...
3. [축 3] ...

**살짝 갈린다 (또는 사주에만 있다)**
4. [축 4] ...

---

**종합**

[한 문장. 짝의 그림 + 시기 가드까지]
```

**일치도 라벨 5단계**
- 🟢🟢🟢🟢🟢 완전 일치 (5/5): 모든 축에서 같은 사람 가리킴
- 🟢🟢🟢🟢⚪ 강한 일치 (4/5): 핵심 축 일치, 1축 미세 차이
- 🟢🟢🟢⚪⚪ 부분 일치 (3/5): 큰 방향 같지만 디테일 갈림
- 🟢🟢⚪⚪⚪ 약한 일치 (2/5): 일부 영역만 일치
- 🟢⚪⚪⚪⚪ 갈림 (1/5): 다른 사람 가리킴
- ⚪⚪⚪⚪⚪ 완전 다른 좌표 (0/5)

**3. 사용자가 "아니" 하면 vault 데이터로만 답**

위 형식에서 "사주 풀이로 보면" 섹션 빼고 출력. 짧은 한 줄: *"사주 보고 싶으면 '사주로도 봐줘' 던지면 추가로 보여준다."*

**4. 카테고리별 vault 문서 매핑**

- 연인 유형 → `love_style.md` + `relationship_protocol.md` + `self_profile.md`
- 직업 → `career_style.md` + `self_profile.md` (시장 자산) + `philosophy.md`
- 사이드 → `side_project_strategy.md` + `self_profile.md` 위험 신호
- 돈 → `investment_style.md` + `philosophy.md` (단계화) + `self_profile.md` 형성 사건
- 친구·동료 → `self_profile.md` 위험 신호·검증된 회복 사례
- 가족 → `self_profile.md` 형성 사건·기본 정보

**5. 카테고리별 검증 레퍼런스 (필요 시만)**

- 연인 유형 → 애착 분포 (Bartholomew 1991), Gottman 결혼 만족도 4 패턴
- 직업 → Holland RIASEC 6 유형, VIA 24 강점 매핑
- 돈 → 한국 30대 자산 분포 (NABO), 행동경제학 손실 회피 (Kahneman & Tversky)
- 가족 → ACE 어린시절 역경 (Felitti et al. 1998)

**6. 사주·별자리 사용 룰 (절대 룰 재확인)**

- 사주·별자리는 vault 데이터를 **덮어쓰지 않는다**. 항상 vault 답변이 본체, 사주는 비교 노트
- 사주 풀이는 `saju.md` 박제 데이터에서만 끌어온다. LLM이 8자·십신·신살을 직접 추정하지 않는다
- 사주가 vault 답변과 갈리면 **사주가 틀린 게 아니라 다른 좌표계**라는 톤으로
- "사주에 약하니 이렇게 해야 한다" 식 처방 ❌. 처방은 vault 자체 약점에서만 도출
- 사이드처럼 사주 전통에 대응 카테고리 없는 영역에서는 사주 풀이 강제로 끌어붙이지 않는다

### 메뉴 다음 안내

사용자가 한 카테고리 풀이를 받은 후에는 자연스럽게 다음 메뉴 또는 자유 질문으로 이어진다. AI는 끝마다 *"다른 영역도 볼래?"* 한 줄로 메뉴 다시 띄울 수 있다. 단 강제 안 함.

루틴 가이드(매일 자문·갱신 주기·백업)는 README·QUICKSTART에 이미 있으니 보고서·카테고리 풀이 끝에 다시 붙이지 않는다.
